**what currently exists?**
A full-stack sports facility booking application named Horizon. It features a FastAPI backend, a React frontend (with Shadcn UI), a MongoDB database, and Redis for slot locking.

Key Features:
-   **Multi-Role Authentication:** JWT-based auth for Player, Venue Owner, Coach, and a Super Admin.
-   **Super Admin Console:** A dedicated dashboard for the SaaS owner to manage users, approve/reject venue owners, and configure platform-wide settings like payment gateway credentials (Razorpay), booking commissions, and SaaS subscription plans.
-   **Core Booking Flow:** Users can discover venues, view availability, and book slots. The booking process is integrated with a real Razorpay payment gateway, which is dynamically configured from the Super Admin settings.
-   **Concurrency Control:** Redis-based distributed locking prevents double bookings.
-   **SaaS Subscriptions:** Venue owners are subscribed to plans (e.g., Free, Basic, Pro) which enforce limits (e.g., number of venues they can create). Owners can view and manage their plan from their dashboard.
-   **Mercenary Marketplace:** A feature for players/teams to post requests for fill-in players (mercenaries). Posts are linked to specific bookings, and hosts can accept applicants, triggering a Razorpay payment flow to hire the mercenary.
-   **Notifications:** An in-app notification system (bell icon in the navbar) alerts users to events like mercenary application acceptance.
